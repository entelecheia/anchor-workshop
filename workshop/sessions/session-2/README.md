## 핵심 개념: 프롬프트 4요소

좋은 프롬프트는 다음 4가지를 포함합니다:

| 요소 | 설명 | 예시 |
|------|------|------|
| **역할** | Claude가 어떤 전문가처럼 행동할지 | "대학 행정 보고서 전문 편집자로서" |
| **맥락** | 배경 정보 | "RISE 사업단 1차년도 성과 보고서입니다" |
| **지시** | 구체적인 요청 | "아래 내용을 개조식으로 바꿔주세요" |
| **형식** | 출력 형태 | "항목당 15자 이내, 명사형 종결" |

## 실습 내용

### 실습 1: 기본 대화
- `exercises/report-draft.md` 파일을 열어 내용 확인
- Claude에 붙여넣기 후 "3줄로 요약해줘" 요청
- 결과 이터레이션: "더 간결하게", "핵심 수치 포함해서"

### 실습 2: 개조식 변환
- 같은 파일을 "개조식 보고서 형식으로 변환" 요청
- 프롬프트 4요소를 모두 포함하여 재시도
- 두 결과 비교 → 어느 쪽이 더 나은지 토론

### 실습 3: 나만의 첫 프롬프트
- 각자 실제 업무에서 반복하는 텍스트 변환 작업 1개 선택
- Claude로 처리 후 Slack `#anchor-workshop`에 결과 공유
