# Slack 사용 문화 가이드

> RISE 사업단 Slack 워크스페이스 운영 원칙

---

## 왜 이 가이드가 필요한가?

Slack은 강력한 도구이지만, 사용 문화가 잘 정착되지 않으면 카카오톡보다 더 혼란스러울 수 있습니다. 이 가이드는 모두가 편안하게 쓸 수 있는 Slack 문화를 만들기 위한 것입니다.

---

## 스레드 사용 (가장 중요)

**원칙: 채널 메시지에 대한 답글은 반드시 스레드로**

채널에 바로 답장하면 대화가 섞여 읽기 어려워집니다.

| 상황 | 하지 말 것 | 할 것 |
|------|-----------|-------|
| 공지에 답변 | 채널에 "확인했습니다" | 스레드에 ✅ 이모지 반응 |
| 질문에 답변 | 채널에 새 메시지로 답변 | 원래 질문의 스레드에 답변 |
| 관련 파일 추가 | 채널에 별도 업로드 | 원래 메시지 스레드에 업로드 |

예외: 새로운 화제를 시작할 때는 채널에 직접 올립니다.

---

## 이모지 반응 문화

짧은 "네", "확인", "감사합니다" 대신 이모지 반응을 사용하면 채널이 훨씬 깔끔해집니다.

| 이모지 | 의미 | 사용 상황 |
|-------|------|----------|
| ✅ | 확인했습니다 / 완료 | 공지 확인, 할 일 완료 |
| 👀 | 보고 있습니다 / 확인 중 | 검토 중인 파일, 처리 중 |
| 👍 | 좋아요 / 동의합니다 | 아이디어, 제안에 동의 |
| 🙏 | 감사합니다 | 도움받은 경우 |
| ❓ | 질문 있습니다 | 내용이 불명확할 때 |
| 🔥 | 훌륭합니다 | 좋은 성과, 우수 사례 |

## 업무시간 외 알림

**원칙: 받는 사람의 퇴근 시간을 존중합니다**

- 야간·주말 메시지: 보내는 건 OK, 답장 기대 금지
- 알림 설정: 각자 근무 시간에 맞게 알림 시간 설정 (`설정 → 알림 → 알림 일정`)
- 급한 경우: 전화 또는 문자 사용 (Slack은 실시간 커뮤니케이션 도구가 아닙니다)

**알림 시간 설정 방법**:
1. Slack 좌측 하단 이름 클릭
2. "환경설정" → "알림"
3. "알림 일정 허용 시간" 설정 (예: 평일 09:00-18:00)

## DM vs 채널 사용 기준

| 상황 | DM | 채널 |
|------|----|----|
| 개인적인 업무 확인 | O | |
| 팀 전체에 관련된 공지 | | O |
| 질문 (다른 사람도 알면 좋은 것) | | O (공개 질문) |
| 민감한 인사 관련 | O | |
| 파일 공유 (팀 활용 가능) | | O |
| 칭찬·감사 | | O (`#random`) |

**원칙**: "나만 알아도 되는 것"은 DM, "팀이 알면 좋은 것"은 채널

## 파일 공유 규칙

- **Google Drive 링크 우선**: 파일을 직접 업로드하지 말고 Drive 링크 공유 (버전 관리 용이)
- **파일 직접 업로드 허용**: 즉시 공유가 필요한 소용량 파일 (이미지, 간단한 문서)
- **개인정보 포함 파일 금지**: 학생 명부, 급여 관련 파일 Slack 공유 불가
- **파일명**: 날짜 + 내용 포함 (예: `260413-예산현황-1분기.pdf`)

## 메시지 작성 팁

- **제목 없는 긴 메시지 지양**: 내용이 길면 첫 줄에 핵심 요약 + 상세 내용 순서
- **멘션(@) 남용 금지**: 정말 그 사람의 응답이 필요할 때만 사용
- **`@channel`, `@here` 최소화**: 긴급 공지 외에는 사용하지 않음
- **오타 수정**: 메시지 위에 마우스 올리기 → "Edit message" (3분 이내 수정 권장)
