## 프롬프트 기여 방법

### 형식 규칙

모든 프롬프트 파일은 다음 4개 섹션을 포함해야 합니다:

```markdown
## 용도
(이 프롬프트가 해결하는 업무 문제)

## 프롬프트
(실제 프롬프트 — {변수}는 중괄호로 표시)

## 사용 예시
(입력 샘플과 출력 샘플)

## 팁
(더 좋은 결과를 위한 추가 조언)
```

### PR 기여 절차

1. 저장소 Fork
2. `prompt-library/{카테고리}/` 아래 새 `.md` 파일 생성
3. 파일명: 기능을 설명하는 영문 소문자 + 하이픈 (예: `quarterly-summary.md`)
4. 위 형식에 맞게 작성
5. PR 제목: `[prompt] {카테고리} — {프롬프트 이름}`
6. PR 제출 → 운영팀 리뷰 후 merge

### 기여 기준

- 실제 업무에 적용하고 효과를 확인한 프롬프트만 기여
- 개인정보·미공개 정보가 없는 것 확인
- `{placeholder}` 형태로 변수 처리 (고유명사 하드코딩 금지)
